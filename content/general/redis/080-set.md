Redis 的 `Set` 结构与 Java 的 [`HashSet`]({{% sref "java-util-hash-set" %}}) 类似，可以看作 `value` 恒为 `null` 的 [`HashMap`]({{% sref "java-util-hash-map" %}})。

- 无序
- 元素不能重复
- 查找快
- 支持交集、并集、差集等集合操作

## `Set` 类型的常见命令

- `SADD key member [member ...]`：向 `set` 中添加一个或多个元素
- `SREM key member [member ...]`：移除 `set` 中的指定元素
- `SCARD key`：返回 `set` 中元素的个数
- `SISMEMBER key member`：判断一个元素是否存在于 `set` 中
- `SMEMBERS key`：获取 `set` 中的所有元素
- `SINTER key [key ...]`：求多个 `key` 的交集
- `SDIFF key [key ...]`：求多个 `key` 的差集
- `SUNION key [key ...]`：求多个 `key` 的并集

```
2) "d"

4) "c"
127.0.0.1:6379> sdiff s1 s2
1) "a"
127.0.0.1:6379> sdiff s2 s1
1) "d"
```
