## `key` 的结构

在 Redis 中，对于每种实体，一般都会以它的 `id` 作为 `key` 存储它，然而 Redis 是键值型数据库，没有表，那么应该如何区分不同实体的 `id` 呢？

Redis 的 `key` 允许多个单词形成层级结构，用 `:` 作为层级分隔符，例如：

```
项目名:业务名:实体名:标识符
```

标识符前面的 3 个单词都是标识符的“前缀”。这样，通过前缀形成层级结构，就能区分不同层级的不同实体。

绝大多数 Redis 的 GUI 客户端都会识别 `key` 的层级结构，并显示成文件夹层级的形式，前缀是文件夹，标识符是文件。

## `value` 的结构

如果 `value` 是一个 Java 对象，则可以将对象序列化为 JSON 字符串存储：

|       `key`       |                `value`                 |
| :---------------: | :------------------------------------: |
|  `heima:user:1`   |   `{"id":1,"name":"Jack","age":21}`    |
| `heima:product:1` | `{"id":1,"name":"手机A","price":4999}` |
