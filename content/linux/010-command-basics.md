Linux 命令的通用格式：`command [-options] parameters`，`命令 [-选项] 参数`（中括号里面的是可选项，可写可不写）

## Linux 目录命令

- `ls [-a -l -h] 目录`：list，列出指定目录下的文件和子目录
  - `-a -l -h` 选项：`-a` 列出包括隐藏路径的所有一级子路径，`-l` 按显示详细信息的列表形式打印，`-h` 显示带单位前缀的路径大小 (大小较大时)
  - Linux 的最小数据计量单位是字节，往大了采用二进制前缀，即 1K=1024
  - 选项 `-a -l -h` 可以随意组合使用，如 `-al`、`-la`、`-lah`，这是通用特性
  - `目录`参数：不指定时采用当前工作目录
  - `.` 是当前目录，`..` 是上级目录，`~` 是当前用户的 HOME 目录

- `cd 目录`：change directory，改变当前工作目录为指定目录，没有选项，不指定`目录`参数时采用`~`

- `pwd`：print work directory，打印工作目录，没有选项和参数

- `mkdir [-p] 目录`：make directory，创建目录，选项 `-p` 是如果父级目录不存在就先创建父级目录，不指定时只能创建一级目录

## Linux 文件命令

- `touch file`：新建文件，如果文件已存在，那就真的只是碰一下

- `cat file`：查看文件，打印文件内容 (concatenate = 拼接，cat 的名字来源于另一个拼接文件的用法)

- `more file`：分页查看大文件，空格翻页，q 退出

- `cp [-r] src dest`：复制路径，`-r` 是递归复制目录中的多级路径，`src` 是源路径，`dest` 是目标路径
  - 即使是空的目录，Linux 也要求显式指定 `-r` 来处理

- `mv src dest`：移动路径，如果 `src` 和 `dest` 的父级目录是同一个，那就相当于重命名源路径

- `rm [-r -f] paths`：删除路径，支持不限个数的 `path` 参数，将删除所有的指定路径
  - `-r` 删除目录，`-f`=force 强制删除 (没有二次确认)
  - 普通用户删除内容不会弹出提示，root 管理员用户删除路径时会有提示
  - 普通用户一般用不到 `-f` 选项
  - rm 命令支持通配符 `*`，用来做模糊匹配
    - 符号 `*` 表示通配符，匹配任意内容 (包含空内容)
    - `test*` 表示匹配任何以 `test` 开头的内容，\*test `表示匹配任何以` test `结尾的内容，*test*` 表示匹配任何包含 `test` 的内容

## Linux 查找命令

- `which 指定命令`：查看指定命令的程序文件存放在哪里
  - Linux 命令的本体就是一个个的二进制可执行程序，和 Windows 系统中的.exe 文件是一个意思
- `find 起始路径 -name "文件或目录名"`：按文件或目录名查找文件
  - 文件或目录名支持通配符，不用通配符会查找与指定名称完全一致的文件或目录
  - 指定名称中没有特殊字符如通配符、空格符时，可以不写引号
- `find 起始路径 -size +|-n[kMG]`：按文件大小查找文件或目录
  - `+|-` 大于或小于，`n` 大小的数字，`kMG` 大小的单位前缀
  - 不写 `+|-` 时，`find` 查找指定大小的文件或目录
  - 查找当前用户 HOME 目录`~` 下小于 10kB 的文件或目录：`find ~ -size -10k`
- `grep [-n] "关键字" 路径`：从结果中通过关键字过滤行
  - 选项 `-n`：可选，在结果中显示匹配的行的行号
  - 参数`关键字`：必填，用于过滤的关键字
  - 参数`文件路径`：必填，要过滤内容的文件路径，可作为内容输入端口
- `wc [-c -m -l -w] 路径`：统计文件的行数、单词数、字节数等
  - `-c` 统计字节数，`-m` 统计字符数，`-l` 统计行数，`-w` 统计单词数，默认统计 `-wlc`
  - `路径`是要统计的文件或目录，可作为内容输入端口

## Linux 管道符

管道符 `|` 将它左边命令的结果，作为右边命令的输入，有内容输出端口的命令都可以放在左边，有内容输入端口的命令都可以放在右边。

- `ls -al ~ | grep "test"`：过滤出 HOME 目录下名称中包含 test 的文件或目录
- `ls -a | wc -l`：统计当前目录下有多少一级文件或目录 (ls 的结果看似是制表符分割的，但是这是自动排版的结果，底层的字符流还是换行符 LF 分割的)
- `cat test.txt | wc -m`：统计 test.txt 的字符数
- `cat test.txt | grep "arch" | grep "linux"`：过滤出既包含 arch 又包含 linux 的行

## Linux 命令行命令

- `echo 输出的内容`：在命令行内输出指定内容，只有一个参数

- 反引号 (\`) 包裹的内容会作为命令执行，并返回执行结果：

  ```bash
  [ror@arch ~]$ echo `pwd` #pwd的结果作为echo的输入
  /home/ror
  ```

- 重定向符 `>` 或 `>>`：`命令 > 文件` 或 `命令 >> 文件`

  - `>` 将左侧命令的结果，覆盖写入到符号右侧指定的文件中
  - `>>` 将左侧命令的结果，追加写入到符号右侧指定的文件中

- `tail [-f -num] 文件`：查看文件尾部内容，跟踪文件的最新更改

  - `-f` 持续跟踪，`-num` 查看尾部 `num` 行，不填默认 10 行
