类是一种数据结构，是一种数据类型，代表现实世界中的“种类”。

> 类是一种数据结构，它可以包含数据成员（常量和字段）、函数成员（方法属性、事件、索引器、运算符、实例构造函数、静态构造函数和析构函数）以及嵌套类型。类类型支持继承，继承是一种机制，它使派生类可以对基类进行扩展和专用化。——《C#语言规范》

## 面向对象的实现风格

* 基于类的（class-based）面向对象实现：C#、C++、Java
* 基于原型的（prototype-based）面向对象实现：JavaScript

## 类的声明

* 类的声明位置：
	* 名称空间内
	* 名称空间外：看似在名称空间外，实则仍在全局名称空间内。不推荐这么做。
	* 类内：嵌套类=成员类
* 声明 declare 即 定义 define（C#与Java）：声明和定义必须连在一起，分不开。
* 声明非定义（C/C++）：声明在 .h 文件，定义在 .cpp 文件。声明 和定义可以合并，但不推荐这么做。

## 类的成员

C#类型包括类和结构体，组成它们的成员有：

* 数据成员
	* 字段
	* 常量
* 函数成员
	* 方法
	* 属性
	* 事件
	* 索引器
	* 运算符
	* 实例构造函数
	* 静态构造函数（只能初始化静态成员，不能有参数）
	* 析构函数
* 成员类（嵌套类）

### 字段 Field

**字段**是与对象或类型关联的变量。

* 与对象关联的字段称为**实例字段**，与类型关联的字段称为**静态字段**。
	* 实例字段的初始化时机是实例创建时，静态字段的初始化时机是运行环境加载该类型时。
	* 实例构造器每次创建实例都会执行，静态构造器只在加载类型时执行一次。
* 字段的修饰符：new、public、protected、internal、private、static、readonly、volatile。

### 属性 Property

**属性**是一种用于访问对象或类型的特征的成员，特征反映了对象或类型的状态。

* 属性是字段的自然扩展。
	* 从命名上看，field更偏向于实例对象在内存中的布局，property更偏向于反映现实世界对象的特征。
	* 对外：暴露数据，数据可以是存储在字段里的，也可以是动态计算出来的。
	* 对内：保护字段不被非法值"污染"。
* 属性由Get/Set方法对进化而来。
* 访问器 getter 和 setter：public int myInt { get{} set{} }
* Setter 中的 value 关键字是上下文关键字。
* Getter 和 setter 可以给访问权限修饰符。private set {} 意味着属性只能在成员方法中赋值。
* 又一个"语法糖”——属性背后的秘密

属性的声明：

* 完整声明：后台（back）成员变量与访问器（注意使用code snippet和refactor工具）
* 简略声明：只有访问器（查看IL代码）
* 动态计算值的属性
* 注意实例属性和静态属性
* 属性的名字一定是名词
* 只读属性：只有getter没有setter
	* 尽管语法上正确，但几乎没有人使用"只写属性”，因为属性的主要目的是通过向外暴露数据而表示对象/类型的状态。
* Getter vs 等效的成员方法：如果属性访问频繁，用成员方法找个合适的时机先算好值存到对象里；如果属性访问不频繁，用 getter 更简洁高效。

VS 快速声明属性：

* **propfull** 加两下 Tab 可以快速完整声明一个属性。
* **prop** 加两下 Tab 可以快速简略声明一个属性。

### 属性与字段的关系

* 一般情况下，它们都用于表示实体（对象或类型）的状态。
* 属性大多数情况下是字段的**包装器（wrapper）**。
* **建议：**永远使用属性来暴露数据（而不是字段）。
	* 属性一般是public，字段永远都是private或protected。
* 字段一般采用驼峰命名法，属性一般采用 Pascal 命名法。

## 常量 Constant

**常量**是表示常量值（可以在编译时算得的值，拿值替换标识符）的类成员。

* 常量隶属于类型而不是对象，即没有“实例常量”。实例常量的角色由只读实例字段来担当。
* 注意区分成员常量（属于类）与局部常量（属于方法）。

常量的声明：const

常量“只读”的应用场景：

* 为了提高程序可读性和执行效率——**常量**
* 为了防止对象的值被改变——**只读字段**
* 向外暴露不允许修改的数据——**只读属性**（静态或非静态），功能与常量有一些重叠
* 想设为常量的值的类型不能被常量声明接受（类、自定义结构体）——**静态只读字段**
