## 隐式转换 (自动转换)

![Java 基本数据类型的取值范围](https://img.xicuodev.top/2026/02/f1c2108a0e96c7299fd0e8ace3329dd6.png "Java 基本数据类型的取值范围")

- 数值类型的隐式转换：看取值范围大小，而非内存占用大小。只有大范围完全覆盖小范围时，才能隐式转换。因为范围覆盖，宽内存的 `long`8 也可隐式转换为窄内存的 `float`4。
  - `byte`1 → `short`2, `char`2 → `int`4 → `long`8 → `float`4 → `double`8
  - 整数都能隐式转 `int`，浮点数和整数都能隐式转 `double`。
  - 整数表达式默认转 `int`，有 `long` 转 `long`；浮点数表达式默认转 `double`。
- 引用类型的隐式转换：
  - 两种数据类型兼容
  - 向上转型 (Upcasting)
  - 装箱
  - `String.valueOf` 方法和 `<numeric-type>.parse<some-type>` 方法

### 符号扩展与无符号扩展 Sign Extension & Zero Extension

当一个窄类型通过隐式转换扩展为一个宽类型时，需要填充窄类型空缺的高位比特。对此有两种扩展方式：符号扩展和无符号扩展。

- **符号扩展**是根据符号位决定填充 `0` 还是 `1` 的比特值​：如果原始值的最高位是 `0`​(正数)，则高位填充 `0`；如果原始值的最高位是 `1`​(负数)，则高位填充 `1`。有符号整数转宽类型，就是采用符号扩展。
- **无符号扩展**是在高位直接填充 `0`，因此也叫 “**零扩展**”(zero extension)。无符号整数 (如 `char`，也就是 `u16`) 转宽类型，就是采用无符号扩展。

> [!tip] 计算机采用补码来表示整数
>
> 在保持数值不变的前提下，将较窄的有符号整数转换为较宽的类型时，计算机会采用符号扩展来填充空缺的高位比特。这是因为计算机采用**补码**来表示整数，如 (为方便书写，用十六进制表示二进制)：
>
> - -32767 `short`：原码 `0xffff`, 补码 `0x8001`
> - -32767 `int`：原码 `0x8000_7fff`, 补码 `0xffff_8001`
>
> 补码和原码的尾数按位取反再加 1 可以互相转换。特别地，将原码 `-0` 看作进位溢出，符号位 `1` 既表示负号，又表示 $1\times2^{n-1}$ ($n$ 是位数)。可知，为保持扩展位数后数值不变，应当采用符号扩展。

## 显式转换 (强制转换)

显式转换无视内存溢出、精度丢失等风险，不惜截断溢出部分也要把数据转换成目标类型。

- 基本类型的显式转换：`(SomeType) someVar`
  - 大范围转小范围
  - `int` 做除法结果也是 `int`，因此会截断小数部分，需要把其中一个操作数显式转换为 `double`：`1/2`=`0`，`1/(double)2`=`0.5`。
- 引用类型的显式转换：
  - 向下转型 (Downcasting，失败抛 `ClassCastException`)
  - 拆箱

### 大端序 Big-Endian

大端序：高位字节 (MSB, Most Significant Byte) 存储在低地址，低位字节 (LSB, Least Significant Byte) 存储在高地址。与人类阅读书写习惯相同。

- Java 的基本数据类型采用大端序来存储。
- JVM 规范：强转导致内存溢出时，截断高位字节，保留低位字节。根据大端序，截断低地址，保留高地址。
