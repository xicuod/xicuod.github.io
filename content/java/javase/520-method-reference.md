title: Java 方法引用
---

方法引用是把已经有的方法拿过来用，把它的方法体当做函数式接口中抽象方法的实现。

- 以下讨论的抽象方法是已经被方法引用实现了的抽象方法。

1. 引用处必须是函数式接口。
2. 被引用的方法必须已经存在。
3. 被引用方法的形参和返回值需要跟抽象方法保持一致。
4. 被引用方法的功能要满足当前需求。

方法引用的分类：`::` 是方法引用符

1. 引用静态方法：`类名::静态方法`
2. 引用成员方法：`对象::成员方法`
   1. 引用其他类的成员方法：`其他类对象::方法名`
   2. 引用本类的成员方法：`this::方法名`
   3. 引用父类的成员方法：`super::方法名`
3. 引用构造方法：`类名::new`
   - 抽象方法的参数会传入引用的构造方法中，因此指定类需要有参数列表匹配的构造方法，如接收一个 String 参数返回一个 Student 参数的抽象方法
4. 其他调用方式
   1. 使用类名引用成员方法：`类名::成员方法`，如 `String::substring`
      - 相当于把抽象方法的第一个形参当作主调用者，把其他参数当作传入参数，来调用成员方法
      - 因此要保证类名与抽象方法的第一个形参的类型一致
      - 因此只能使用抽象方法的第一个形参的类型的成员方法
   2. 引用数组的构造方法：`元素类型[]::new`
      - 数组的构造方法是隐藏的，但可以通过平时创建数组对象的表达式 `new String[3]` 看出它是一个接收 `int len` 参数，返回长度为 `len` 的 `String[]` 类型数组的方法
      - 只要是接收 `int` 返回 `T[]` 的 lambda，都能替换成数组的构造方法引用 `T[]::new`
