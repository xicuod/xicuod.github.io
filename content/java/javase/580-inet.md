## 两种常见的网络软件架构

C/S (Client/Server, 客户端/服务器)：在用户本地需要下载并安装客户端程序，在远程有一个服务器端程序。

B/S (Browser/Server, 浏览器/服务器)：只需要一个浏览器，用户通过不同的网址访问不同的服务器。

两种架构的优缺点：

- B/S 架构：
  - 不需要开发客户端，只需要开发浏览器前端和服务器后端
  - 用户不需要下载，打开浏览器就能使用
  - 如果应用过大，用户体验受到影响
- C/S 架构：
  - 画面可以做的非常精美，用户体验好
  - 需要开发客户端，也需要开发服务端
  - 用户需要下载和更新的时候太麻烦

## 网络通信三要素

IP 地址：设备在网络中的地址和唯一标识。IP 地址与计算机的网卡绑定，用于确定对方计算机在互联网上的地址。

端口号：应用程序在设备中唯一的标识，两字节整数，取值范围 `0`-`65535`。一个端口号只能一个应用程序使用。`0`-`1023` 的端口号用于一些知名的网络服务或应用，我们使用 `1024` 以上的就可以了。

协议：连接和通信的规则，常见的协议有 UDP、TCP、HTTP、HTTPS、FTP。

## IPv4 和 IPv6

IPv4 (Internet Protocol version 4, 互联网通信协议第四版) 采用 32 位地址长度，分成 4 组，点分十进制表示

- IPv4 地址举例：`11000000 10101000 00000001 01000010` = `192.168.1.66`
- 32 位二进制只能提供 2^32=4,294,967,296 (不到 43 亿) 个 IPv4 地址，不够用
- 2019 年 11 月 26 日，所有 IPv4 地址全部分配完毕

IPv6 (Internet Protocol version 6, 互联网通信协议第六版) 采用 128 位地址长度，分成 8 组，冒分十六进制表示。

- IPv6 地址举例：`2001:DB8:0:23:8:800:200C:417A` (省略高位补位的 0)
- 0 位压缩表示法：`FF01:0:0:0:0:0:0:1101` = `FF01::1101` (省略中间的 0 和冒号)
- 由于互联网的蓬勃发展，IP 地址的需求量愈来愈大，而 IPv4 的 IP 总数是有限的，于是推出了 IPv6
- 128 位二进制可以提供 2^128=3.403e38 (约 340 兆兆兆) 个 IPv6，可以为地球上的每一粒沙子都分配一个 IPv6 地址

## IPv4 地址分类

在 IPv6 尚未普及的过渡时期，为节省 IP，将 IPv4 分为万维网使用的公网地址和局域网使用的私有地址。

- `192.168.` 开头的就是私有地址，范围 `192.168.0.0`-`192.168.255.255`，专门为组织机构内部使用
- 一般是路由器拥有公网 IP，再让路由器给每个上网设备分配局域网 IP，来让这些设备共享同一个公网 IP

特殊 IP 地址：

- `127.0.0.1` (`localhost`)：是回送地址 (本地回环地址, 本机 IP)，永远只会寻找当前所在本机。
  - 如果用路由器给当前主机分配的局域网 IP，那么数据要先发往路由器，路由器再转发回来，并且换个路由器这个地址可能就不一样了，而本地回环地址的数据在经过当前主机的网卡时就会发回来。

Windows 下常见的有关网络的 cmd 命令：

- `ipconfig`：检查本机 IP 地址，macOS 上可以用 `ifconfig`
- `ping`：检查网络是否连通

## 网络通信协议模型：OSI 参考模型和 TCP/IP 参考模型

计算机网络中，连接和通信的规则被称为网络通信协议。

- OSI 参考模型：世界互联协议标准，全球通信规范，因模型过于理想化，未能取得广泛应用。
- TCP/IP 参考模型 = TCP/IP 协议：事实上的国际标准。

### 两个重要的传输层协议：UDP 和 TCP

UDP (用户数据报协议, User Datagram Protocol)：UDP 是面向无连接的通信协议。

- 速度快，有大小限制，一次最多发送 64K，数据不安全，易丢失数据。
- 面向无连接：尽最大努力交付，不管是否已经连接成功，能收到就收到，收不到就拉倒。
- 适用于容许丢失少量数据的即时应用场景，语音、视频通话，直播，在线播放视频。

TCP (传输控制协议, Transmission Control Protocol)：TCP 是面向连接的通信协议。

- 速度慢，没有大小限制，数据安全。
- 面向连接：确保连接成功才会发送数据。
- 适用于不容许丢失任何数据的场景，上传、下载文件，电子通信，电子邮件。

### UDP 的三种网络通信方式

- 单播：一对一
- 组播：一对多
  - 组播地址：`224.0.0.0`-`239.255.255.255`，其中 `224.0.0.0`-`224.0.0.255` 为预留的组播地址
  - 一个组播地址可表示多台上网设备
- 广播：一对全，广播地址 `255.255.255.255`

### TCP 的三次握手、四次挥手

- 三次握手：确保连接建立
- 四次挥手：确保连接断开，且数据处理完毕

## `java.net.InetAddress`

`InetAddress` 是用来表示 IP 地址的类，`Inet` 就是 Internet，`Inet4Address` 是 IPv4，`Inet6Address` 是 IPv6。

`InetAddress` 方法：

- `static InetAddress getByName(String host)` 确定主机名称的 IP 地址
  - 主机名称可以是机器名称，也可以是 IP 地址
- `String getHostName()` 获取此 IP 地址的主机名
- `String getHostAddress()` 返回文本显示中的 IP 地址字符串

## `java.net.DatagramSocket` (UDP)

通过 `java.net.DatagramSocket` 使用 UDP 发送数据：

- 创建发送端的 `DatagramSocket` 对象
- 数据打包：创建 `DatagramPacket` 对象
- 发送数据
- 释放资源

`DatagramSocket` 构造方法：构造时会绑定端口，之后就是通过这个端口往外发送数据。

- 空参：从所有可用的端口中随机选取一个
- 有参：指定端口号绑定

通过 `java.net.DatagramSocket` 使用 UDP 接收数据：

- 创建接收端的 `DatagramSocket` 对象
- 接收数据包：接收 `DatagramPacket` 对象
- 解析数据包
- 释放资源

在接收的时候，一定要绑定端口，而且绑定的端口一定要跟发送的端口保持一致。

`DatagramSocket` 成员方法：

- `send()` 方法：接收一个数据包参数，从指定端口发出指定的数据包
- `receive()` 方法：接收一个数据包参数，接收指定端口发来的下一个数据包
  - `receive()` 方法是阻塞的，线程运行到这里会无限等待，直到指定端口发来下一个数据包

UDP 实现单播、组播和广播：

- UDP 单播：使用 `DatagramSocket`
- UDP 组播：使用 `MulticastSocket`
  - 发送端：目标 IP 要指定组播地址
  - 接收端：要用 `MulticastSocket` 对象的 `joinGroup` 方法把当前主机添加到指定的组播地址
- UDP 广播：把单播发送端的目标地址改为广播地址即可

## `java.net.Socket` (TCP)

TCP 通信协议是一种可靠的网络协议，它在通信的两端各建立一个 `Socket` 对象，通信之前要保证连接已经建立，通过 `Socket` 对象产生 I/O 流来网络通信。

Java 客户端通过 `java.net.Socket` 使用 TCP 发送数据：

- 创建客户端的 `Socket` 对象与指定服务器连接：`Socket(String host, int port)`
  - 在创建对象的同时会连接服务端，如果连接不上，代码会报错
- 获取 socket 的输出流写出数据：`OutputStream getOutputStream()`
- 释放资源：`void close()`

Java 服务端通过 `java.net.Socket` 使用 TCP 接收数据：

- 创建服务器的 `ServerSocket` 对象：`ServerSocket(int port)`
- 监听客户端连接，返回一个 `Socket` 对象：`Socket accept()`
- 获取 `socket` 的输入流读取数据，并把数据显示在控制台：`InputStream getInputStream()`
  - 这一步获取的 `InputStream` 对象的 `read` 方法会从 socket 连接通道中读取数据。但是，需要有一个结束标记，`read` 方法才会返回 `-1`。否则，程序就会一直阻塞在 `read` 方法这里，等待读取下面的数据。
  - 发送方用 socket 的 `shutdownOutput()` 方法关闭发送方向 (但不关闭输出流)，这会使 TCP 发送一个 `FIN` 包，来告知接收方发送完毕，相应地，接收方输入流的 `read` 方法会返回 `-1`
- 释放资源：`void close()`
  - 连接途中不要手动关闭 socket 的 I/O 流，它们会随最后的 `socket.close()` 一同关闭。

选用合适的 I/O 流：

- 传输非纯文本文件或密文时，用字节缓冲流包裹 socket 的字节流，提高效率
- 传输明文时，用转换流包裹 socket 的字节流，使之成为字符流，然后再用字符缓冲流包裹，提高效率
