`TreeSet` 特点：可排序、不重复、无索引

- 可排序：按照元素的默认规则 (从小到大) 自动排序。

`TreeSet` 底层：基于[红黑树]()的数据结构实现排序，增删改查性能都较好。

`TreeSet` 遍历：

- 迭代器遍历
- 增强 `for` 遍历
- lambda 遍历

`TreeSet` 的默认排序规则：

- 对于数值类型 (`Integer`、`Double` 等)：默认按照从小到大的顺序排序。
- 对于字符、字符串类型：按照字符在 ASCII 码表中的数字升序排序。
- 对于字符串：逐个字符比较。长短不一时，设定缺位的空字符是最小的。

`TreeSet` 存储自定义类型时，指定排序比较规则的两种方式：

## 默认排序（自然排序）

默认排序就是 [Java Bean]({{% sref "java-bean" %}}) 实现 `Comparable<T>` 接口的 `compareTo` 方法来指定红黑树比较规则。

`compareTo` 方法：

- 关键字 `this`：当前要添加的元素
- 参数 `T o`：已经在红黑树的元素
- 返回值 `int`：
  - 如果是负数，认为 `this` 小，存 `o` 的左孩子
  - 如果是正数，认为 `this` 大，存 `o` 的右孩子
  - 如果是 `0`，认为 `this` 的相同元素已经存在，舍弃

## 比较器排序

比较器排序就是构造 `TreeSet` 对象时，传入比较器 `Comparator` 接口参数，传入一个实现它的实例重写它的 `compare` 方法来指定红黑树比较规则。

`compare` 方法：

- 参数 1 `T o1`：当前要添加的元素
- 参数 2 `T o2`：已经在红黑树的元素
- 返回值 `int` 处理方式同 `Comparable` 接口的 `compareTo` 方法

## 排序方式使用原则

排序方式使用原则：默认使用自然排序，如果不能满足当前需求，就使用比较器排序。

- 自然排序：内部，静态，类所有，使用自己写的类
- 比较器排序：外部，动态，扩展性，使用别人写的类

## 排序方式调用规则

对内置类，只有自然排序时调用自然排序，两个都有时调用比较器排序；对自定义类，只有一个时调用那一个，两个都有时调用比较器排序。
