title: Java 封装
---
类有哪些固有属性，就要封装哪些数据（成员变量），而且还要一并封装操作这些数据的行为（成员方法），而这都是为了“高内聚”。因为成员方法要操作成员变量，它们是紧密关联的，所以必须把它们封装到一起去。如果把它们分开，就会因为它们而导致两个类耦合在一块了。

> [!note] “高内聚”和“低耦合”是紧密相联的
> 
> - 低内聚几乎总是伴随着高耦合，因为职责混乱，导致不同的类互相牵扯。
> - 高内聚是低耦合的必要条件，但不是充分条件。高内聚让模块独立，给低耦合打下了良好的基础。但要实现低耦合，还需要精心设计模块间的交互方式（接口隔离、依赖倒置等）。

类的属性应当是私有的，并且要根据是否可供外部读写提供 `get` 或 `set` 方法。
