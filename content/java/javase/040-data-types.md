## 基本类型 Primitive Types

Java 不是完全面向对象的语言，它更务实，提供了 8 种把数据直接存在栈中的基本类型，同时它们都有自己的包装类 (wrapper class，装拆箱)。

- 数值类型 (numeric-types)：
  - 整数类型：`byte` 1 (取值范围 - 128\~127), `short` 2, `int` 4, `long` 8
  - 浮点类型：`float` 4, `double` 8, `BigDecimal` (金融)
  - 字符类型：`char` 2
- 布尔类型：`boolean` 1b=1 位 = 1 比特

### 数值类型 - 整数和浮点数

- 默认初始值为 0 和 0.0。
- 整数和浮点数字面量的默认类型为 `int` 和 `double`，过大的整数需要显式指定大范围类型。
- 显式指定字面量类型：`1_0000_0000_0001L` 为 `long`，`1.01F` 为 `float`，`1.0001D` 为 `double`。

> [!note] 显式指定字面量类型
>
> 虽然 `L`、`F` 和 `D` 大小写不敏感，但 `L` 推荐大写，防止字体不好导致小写的 `l` 易与数字 `1` 混淆。

- 不同进制的整数字面量：
  - `0b` 或 `0B` 打头：二进制数
  - `0` 打头：八进制数
  - `0x` 打头：十六进制数，十六进制用 A-F 表示十进制的 10-15
- JDK7 起，数值类型字面量的数字之间能随便加下划线 `_` 了，但头尾和小数点旁边不能加。如 `float x = 1_0___11.1_2f;`，中文环境下一般的写法是 `10_0000_0000` 四位一组。

> [!tip] 避免使用浮点数做比较
>
> 浮点数的长度有限，不可能表示小数点后无穷位的数，能表示的数字其实同整数一样是离散的，只不过离散程度小了一点。而除法往往除不尽，会产生无穷位小数。即使是范围大的浮点数，在计算除法时，也必须面临舍入误差。因此，最好完全避免使用浮点数做比较。像银行业务这种需要精确算钱的场景，应该用 `BigDecimal` 数学工具类。

### 数值类型 - 字符

`char` 类型固定占用 2 个字节 (16 位)，Java 采用 UTF-16 编码格式处理字符。

- 1 个 UTF-16 码元 = 2 字节 = 16 位，一个 Unicode 字符的码位为 1 或 2 个码元
  - 1 个 BMP 字符的 UTF-16 码位 = 1 个 UTF-16 码元
  - 1 个辅助平面字符的 UTF-16 码位 = 2 个 UTF-16 码元
  - [Unicode 基本多文种平面 BMP 和辅助平面](http://blog.xicuodev.top/notes/20260222-character-encoding/#unicode%E4%B8%87%E5%9B%BD%E7%A0%81%E6%A0%87%E5%87%86)
- 一个 `char` 宽 2 字节，可以存一个 BMP 字符。
- 辅助平面字符长 2 个 UTF-16 码元 = 4 字节，需要代理对，即两个 char 按字符串存储。

```java
String s = "\uD840\uDC00"; /* 𠀀 */
```

- `char` 的转义写法：`\u` 转义符 + 1 个十六进制 UTF-16 码元，一些特殊符号有简写法
  - `'\u0000'`=`'0'`(`char` 默认初始值)，`'\u0061'`=`'a'`
  - 特殊符号：`\b` 退格，`\n` 换行，`\r` 回车，`\t` 制表，`\'` 单引号，`\"` 双引号，`\\` 反斜杠

## 引用类型 Reference Types

引用类型存的是地址，有类、接口、数组、枚举和注解。

```java
double[] douArr = {/* elems */};
sout(douArr); /* [D@31221be2 */
int[] intArr = new int[3];
sout(intArr); /* [I@404b9385 */
```

- `[` 表示数组，`D` 表示 `double`，`I` 表示 `int`，`@` 表示后面是一个十六进制地址。
- 默认初始值为 `null`。
- 所有引用类型 (除接口外) 都直接或间接地继承 `Object` 类。
- 继承不是类的专利，其他引用类型也有类似的机制。
- 注解是一种特殊的接口，枚举是一种特殊的类。
- 数组是一种低级数据结构，而非类。运行时的数组实例属于动态创建的轻量类。

| 类型 | 继承基类 (基接口)               | 实现接口            | 具体表现                                                                                |
| ---- | ------------------------------- | ------------------- | --------------------------------------------------------------------------------------- |
| 类   | \[o]                            | \[o]                | -                                                                                       |
| 数组 | \[o] 隐式继承 `Object` 类       | \[o] 隐式实现接口   | 所有数组类型隐式继承 Object，并实现 `Cloneable` 和 `java.io.Serializable` 接口          |
| 接口 | \[o] 可继承其他接口             | \[x]                | 接口只能通过 `extends` 继承其他接口 (如 `interface A extends B, C`)                     |
| 注解 | \[o] 隐式继承 `Annotation` 接口 | \[x]                | 所有注解隐式继承 `java.lang.annotation.Annotation` 接口                                 |
| 枚举 | \[o] 隐式继承 `Enum` 类         | \[o] 可显式实现接口 | 枚举隐式继承 `java.lang.Enum`，同时可显式实现接口 (如 `enum Color implements Runnable`) |
